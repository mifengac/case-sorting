你是一名资深 Python 工程师。请帮我开发一个轻量的「笔录扫描件转文本」工具，要求如下，请输出完整可运行的项目代码，不要省略任何文件。

## 背景与环境（硬性约束）

- 部署目标是一台**完全离线（无互联网）的内网服务器**，系统 Ubuntu 22.04，**纯 CPU，没有 GPU**。
- 不允许使用任何需要 `apt` 安装的系统级依赖，所有依赖必须是纯 pip 包。
- 不用 PaddleOCR 本体（paddlepaddle 框架太大），不用 Java。
- 用户是中文使用者，界面和提示信息用简体中文，代码注释用中文。

## 技术选型（已定，不要更改）

- OCR 引擎：`rapidocr_onnxruntime`（RapidOCR，基于 onnxruntime 的轻量 OCR 推理库）。
- 模型：PP-OCRv5 mobile 系列 ONNX 模型，共 3 个文件，放在项目的 `models/` 目录下，启动时从该目录加载，**禁止联网下载模型**：
  - `models/PP-OCRv5_mobile_det.onnx`（文字检测）
  - `models/PP_OCRv4_mobile_cls.onnx`（方向分类）
  - `models/PP-OCRv5_mobile_rec.onnx`（文字识别）
  - 请在代码里把模型路径做成可配置项（如环境变量或 config 文件），并处理好「指定了自定义 det/cls/rec 模型路径时如何传给 RapidOCR」的初始化逻辑。
- PDF 转图片：用 `PyMuPDF`（fitz），逐页按 300dpi 渲染成图片，**不要用 pdf2image/poppler**（离线装不了系统依赖）。
- Web 框架：FastAPI + uvicorn。
- 图片处理：Pillow。

## 功能要求

### 1. 网页服务（主功能）

- 一个简洁的单页面（服务端渲染的 HTML 即可，不要引入前端框架，不要引用任何 CDN 外链资源，因为内网离线）：
  - 拖拽/选择上传 PDF 或图片（jpg/jpeg/png/bmp/webp），支持多文件。
  - 上传后后台识别，页面显示进度或状态（简单的轮询即可）。
  - 识别完成后，逐页显示识别出的文本，提供「一键复制全部」和「下载 .txt」按钮。
- 文件大小限制 50MB，PDF 页数限制 100 页，超限给出友好中文报错。
- 识别是 CPU 密集型，用串行队列逐个处理任务，避免并发把 CPU 打满；任务状态保存在内存里即可（允许重启后丢失）。
- 上传的文件和处理结果保存在 `data/uploads/` 和 `data/results/`，提供结果过期清理逻辑（默认 24 小时）。

### 2. 命令行批处理

- `python -m app.batch <输入目录> -o <输出目录>`：
  - 递归扫描输入目录下的 PDF 和图片，逐个识别，每份文件输出一个同名 `.txt` 到输出目录。
  - 打印每份文件的进度和耗时，单个文件失败不中断整体，最后汇总成功/失败数量。

### 3. 识别后处理

- RapidOCR 返回的是带坐标的文本框列表，请按「从上到下分行、行内从左到右」排序后拼接：同一行的文本框用空格连接，行间换行；纵向距离明显大于行高的地方插入空行分段。目标是让打印体笔录转换后段落顺序正确、可直接阅读。

## 离线部署支持（重要）

在项目里提供两个脚本：

1. `scripts/download_offline_bundle.sh`：在**有网机器**上运行，把依赖 whl 包 `pip download` 到 `wheels/` 目录，并提示用户去 HuggingFace 下载 3 个模型文件放进 `models/`（给出模型文件名和 HuggingFace 搜索关键词即可）。
2. `scripts/install_offline.sh`：在**离线内网机器**上运行，创建 `.venv` 虚拟环境，用 `pip install --no-index --find-links=wheels` 安装依赖，并检查 `models/` 下 3 个模型文件是否存在，缺失则报错退出。

## 项目结构

```

├── app/
│   ├── __init__.py
│   ├── main.py          # FastAPI 入口 + 网页路由
│   ├── config.py        # 配置项（模型路径、限制、目录等）
│   ├── ocr_engine.py    # RapidOCR 封装：初始化、识别单张图片、结果排序拼接

│   ├── tasks.py         # 串行任务队列与状态管理
│   ├── batch.py         # 命令行批处理
│   ├── templates/       # Jinja2 模板（单页面）
│   └── static/          # 少量 css/js，全部本地化
├── models/              # 放 3 个 ONNX 模型（附 .gitkeep 和 README 说明）
├── scripts/
│   ├── download_offline_bundle.sh
│   └── install_offline.sh
├── requirements.txt
├── run.sh               # .venv/bin/uvicorn app.main:app --host 0.0.0.0 --port 8000
└── README.md            # 中文说明：在线打包 → 离线安装 → 启动 → 使用
```
