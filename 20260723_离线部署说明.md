# case-sorting（笔录扫描件转文本）离线部署说明

目标环境：离线内网 Ubuntu 22.04 服务器，纯 CPU。本目录已带齐全部所需文件，**全程不需要联网**。

## 方式一：Docker Compose（推荐，最省事）

前提：内网服务器装有 Docker 及 compose 插件。

```bash
# 1. 导入镜像（bilu-ocr-image.tar.gz 已在本目录）
sudo docker load -i bilu-ocr-image.tar.gz

# 2. 启动（在本目录执行，端口已在 docker-compose.yml 里设为 5013）
sudo docker compose up -d

# 3. 验证
curl http://127.0.0.1:5013/api/health   # 返回 {"status":"ok"} 即正常
sudo docker compose ps                  # STATUS 显示 healthy 即健康检查通过
```

浏览器访问 `http://服务器IP:5013` 即可使用。模型已打在镜像里，无需任何额外文件。
上传文件和识别结果会持久化到本目录的 `data/`。

常用命令：`sudo docker compose logs -f`（看日志）、`sudo docker compose down`（停止）、`sudo docker compose restart`（重启）。
改端口：编辑 `docker-compose.yml` 里 `"5013:8000"` 左边的数字，再 `sudo docker compose up -d`。

也可以不用 compose，直接 docker 命令：

```bash
sudo docker run -d --name bilu-ocr -p 5013:8000 --restart unless-stopped bilu-ocr:latest
```

## 方式二：无 Docker，用 Python 虚拟环境

前提：内网服务器有 Python 3.10（Ubuntu 22 自带）。

```bash
cd case-sorting
bash scripts/install_offline.sh   # 用 wheels/ 离线装依赖，自动校验 models/ 里 3 个模型
bash run.sh                        # 启动，默认 8000 端口
```

如需改端口：`BILU_OCR_PORT=5013 bash run.sh`

## 目录清单

| 内容 | 作用 |
|------|------|
| `app/` | 项目代码 |
| `models/` | 3 个 ONNX 模型（det/cls/rec，共约 21 MB） |
| `scripts/` | 离线安装/打包脚本 |
| `wheels/` | Python 3.10 离线依赖包（方式二用） |
| `bilu-ocr-image.tar.gz` | Docker 镜像包（方式一用） |
| `requirements.txt` / `run.sh` / `Dockerfile` | 部署配套文件 |

## 使用

- 网页：拖拽上传 PDF / 图片 → 识别 → 复制或下载 txt
- 命令行批处理（方式二）：`.venv/bin/python -m app.batch /path/to/笔录目录 -o /path/to/输出目录`
- 命令行批处理（方式一）：`docker exec bilu-ocr python -m app.batch /path/in/container -o /path/out`（需先 `-v` 挂载目录进容器，或改用 `docker cp`）

## 备注

- 单文件 ≤ 50 MB，PDF ≤ 100 页；识别任务串行排队；结果默认保留 24 小时
- 针对打印体笔录；若手写较多，可后续换 server 版模型（只换 models/ 里的文件）
